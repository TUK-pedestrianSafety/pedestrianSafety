프론트엔드 센서 시뮬레이션 좌표 및 거리 계산 로직 설명

1. 좌표계 설정 (Canvas)
   - 전체 크기: 800px (너비) x 400px (높이)
   - 1 픽셀(px)당 실제 거리 비율: 0.3미터 (SCALE_FACTOR = 0.3)
     즉, 화면상 100px 이동 시 실제로는 30m 이동한 것으로 계산됩니다.

2. 센서 배치 로직
   - 센서는 도로(Y=200px)를 기준으로 위쪽(Y=150px)과 아래쪽(Y=250px)에 배치됩니다.
   - 센서 간 간격(SENSOR_SPACING): 120px (실제 거리 36m)
   - 시작 위치: X=100px 지점부터 배치 시작
   - 홀수 ID 센서 (1, 3, 5...): 위쪽 도로변 (Y=150)
   - 짝수 ID 센서 (2, 4, 6...): 아래쪽 도로변 (Y=250)

3. 차량 이동 로직
   - 차량 Y 위치: 200px (도로 중앙)
   - 이동 단위(MOVE_STEP): 한 번 업데이트(틱)마다 20px 씩 이동
   - 이동 속도: 사용자가 설정한 밀리초(ms) 단위로 위치 업데이트
   - 차량이 화면 끝(CAR_END)을 넘어가면 다시 시작 지점(CAR_START)으로 초기화됩니다.

4. 거리 계산 (Distance Calculation)
   - 계산 식: (센서 X 위치 - 차량 X 위치) * 0.3
   - 결과 값의 의미:
     (+) 양수: 차량이 센서에 접근 중 (차량이 센서보다 왼쪽에 있음)
     (-) 음수: 차량이 센서를 지나침 (차량이 센서보다 오른쪽에 있음)
     0: 차량이 센서와 정확히 같은 수직선상에 위치

5. 위험도 판정 (거리 절대값 기준)
   - 위험(Danger): 15m 이내 (화면상 약 50px 이내)
   - 경고(Warning): 15m ~ 25m (화면상 약 50px ~ 83px)
   - 안전(Safe): 25m ~ 50m (화면상 약 83px ~ 166px)
   - 꺼짐(Off): 50m 초과

백엔드 연동 및 움직임 로직과 관련된 파일은 크게 두 곳으로 나뉩니다.

백엔드 데이터 연동 (API 호출):
파일 위치: 
Frontend/src/services/mockApi.ts
설명: 백엔드 API와 통신하여 데이터를 가져오는 역할을 합니다. 현재는 
RiskEvent와 SensorHealth - fetch나 axios를 통해 서버로 요청을 보내도록 수정하면 됩니다.
움직임 시뮬레이션 로직 (프론트엔드 자체 로직):
파일 위치: 
Frontend/src/components/SensorSimulation.tsx

설명: 차량의 움직임과 센서 감지 로직이 구현된 파일입니다. 현재는 백엔드와 연동되지 않고, 프론트엔드 내부에서 setInterval을 사용해 차량 위치를 변경하고 거리를 계산하고 있습니다.
요약하자면:
데이터 구조 및 통신을 수정하려면 
src/services/mockApi.ts

차량 이동 방식이나 센서 반응 로직을 수정하려면 
src/components/SensorSimulation.tsx

만약 SensorSimulation의 움직임 데이터를 백엔드에서 받아오거나 백엔드로 
보내고 싶다면, 
mockApi.ts
에 관련 API 함수를 추가하고 
SensorSimulation.tsx
에서 이를 호출하는 방식으로 연동해야 합니다.
